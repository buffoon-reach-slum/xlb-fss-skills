# Comment First

在编写或修改代码前，先确定需要补充的注释点，并在实现时补上：

- 只为关键业务逻辑、复杂流程、接口约束、非直观的状态/条件添加注释
- 避免给显而易见的语句添加注释
- 注释保持短句、清晰、贴近代码真实行为
- 注释语言与当前对话语言一致

使用场景：

- 用户要求“加注释/补注释/写注释”
- 代码改动涉及关键流程或复杂判断
- 修改或新增模块时需要解释约束或边界条件
